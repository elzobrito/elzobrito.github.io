# T-1000 — Especificação baseline ESAA (site estático)

**Projeto:** elzobrito.github.io  
**Task kind:** spec  
**Status alvo:** review → done  
**Dependências:** nenhuma

## Objetivo

Definir o baseline funcional e de governança ESAA para o site profissional estático hospedado em GitHub Pages, garantindo que futuras tarefas (`T-1010` impl, `T-1020` QA) tenham critérios verificáveis.

## Escopo do sistema

Site single-page em HTML/CSS/JS puro, sem build step, sem backend e sem dependências npm em runtime. Publicação via branch `main` e GitHub Pages (`/` root).

### Artefatos versionados

| Arquivo | Função |
|---------|--------|
| `index.html` | Conteúdo, SEO, Open Graph, seções de navegação |
| `style.css` | Layout responsivo, tema claro/escuro |
| `script.js` | Menu mobile, alternância de tema, ano no rodapé |
| `assets/foto-elzo.jpg` | Foto profissional opcional |
| `README.md` | Instruções de edição e publicação |

## Requisitos funcionais

### R-01 — Conteúdo e navegação

- O site deve apresentar seções: Sobre, Atuação, Consultorias, Formatos, Pesquisa e Contato.
- Navegação interna por âncoras (`#sobre`, `#atuacao`, etc.) com menu acessível em desktop e mobile.
- `lang="pt-BR"` no documento raiz.

### R-02 — SEO e compartilhamento

- `<title>` e `<meta name="description">` descritivos.
- Open Graph: `og:title`, `og:description`, `og:type=website`, `og:url=https://elzobrito.github.io`, `og:locale=pt_BR`.
- `meta name="robots" content="index, follow"`.

### R-03 — Tema e acessibilidade

- Alternância de tema claro/escuro via botão com `aria-label`.
- Menu mobile com `aria-expanded` e `aria-controls`.
- Texto alternativo ou `sr-only` em controles icônicos.

### R-04 — Contato

- E-mail público: `elzobrito@gmail.com`.
- Links para GitHub (`https://github.com/elzobrito`) e LinkedIn (`https://www.linkedin.com/in/elzobrito`).
- Sem formulários que enviem dados a servidores próprios (apenas `mailto:` e links externos).

### R-05 — Compatibilidade GitHub Pages

- Nenhum processo de build obrigatório.
- Paths relativos para CSS/JS/assets.
- Site funcional abrindo `index.html` localmente ou via Pages.

## Requisitos não funcionais

### N-01 — Performance

- Sem frameworks JS; payload total da página < 500 KB (excluindo foto opcional).
- CSS e JS em arquivos únicos minificáveis manualmente se necessário.

### N-02 — Segurança

- Sem scripts de terceiros não documentados.
- Links externos com `rel="noopener noreferrer"` quando `target="_blank"`.

### N-03 — Governança ESAA

- Alterações de conteúdo passam por tasks `spec` → `impl` → `qa` no `.roadmap/`.
- Projeções (`roadmap.json`) não são editadas manualmente.

## Critérios de aceitação

| ID | Critério | Verificação |
|----|----------|-------------|
| CA-01 | Todas as seções de R-01 existem em `index.html` | Inspeção DOM / grep por `id=` |
| CA-02 | Metatags de R-02 presentes no `<head>` | Inspeção `index.html` |
| CA-03 | Botão de tema e menu mobile operam sem erro no console | Abrir localmente + DevTools |
| CA-04 | Contatos de R-04 corretos e clicáveis | Inspeção links `href` |
| CA-05 | `README.md` descreve edição e publicação Pages | Revisão documental |
| CA-06 | Nenhum backend ou API própria no escopo | Ausência de fetch a origens não listadas |
